Overview

The Merchandising Rules API allows merchants to control search results through three types of rules:

  • Boost: Increase/decrease product visibility scores
  • Pin: Place specific products at fixed positions
  • Hide: Completely exclude products from results

Rules are evaluated on every search request and applied before personalization.

Request/Response Schemas

MerchandisingRuleCreate

Field Type Required Default Constraints
name string Required - 1 ≤ length ≤ 255
description string Optional null length ≤ 1000
rule_type string Required - "boost", "pin", "hide"
priority integer Optional 500 1 ≤ priority ≤ 1000
is_active boolean Optional true true, false
conditions object Required - See Conditions section
action_config object Required - See Action Config section

Rule Type Enum Values

"boost" - Increase/decrease product visibility scores "pin" - Place specific products at fixed positions "hide" - Completely exclude products from results

Priority Range

Valid Range: 1 to 1000 (integer)
Default: 500
Higher = More Important: Rules with higher priority take precedence in conflicts

Boolean Values

true - Rule is active / field is enabled false - Rule is inactive / field is disabled

Field Reference

Conditions

Structure:

{
  "type": "<condition_type>",
  "operator": "<operator>",
  "value": "<value>"
}
Condition Type Valid Operators Value Type Description
query_match "exact", "contains" string Match against search query
category "equals", "in" string (equals) or array (in) Match against product categories
product_id "equals", "in" string/int (equals) or array (in) Match against product IDs

Query Match Operators

"exact" - Query must exactly match the value (case-insensitive) "contains" - Query must contain the value as a substring (case-insensitive)

Category/Product ID Operators

"equals" - Must match exactly (single value) "in" - Must match any value in the list (array)

Action Config - Boost

Field Type Required Constraints
boost_factor float Required 0.1 ≤ boost_factor ≤ 10.0
Boost Factor Ranges:
• 0.1 - 0.9: Demote (reduce visibility)
• 1.0: Neutral (no change)
• 1.1 - 10.0: Boost (increase visibility)

Action Config - Pin

Field Type Required Constraints
position integer Required 1 ≤ position ≤ 1000
product_id string Required Non-empty

Action Config - Hide

Empty Object: Hide rules require no fields in action_config. Use {}

Validation Rules

Field Validation

name - Click to expand validation rules
  • Type: String
  • Required: Yes
  • Min Length: 1 character
  • Max Length: 255 characters
  • Pattern: Any UTF-8 string (no regex restriction)
  • Error Messages:
    • "name is required" (if missing)
    • "name must be at least 1 character" (if empty)
    • "name must be at most 255 characters" (if too long)
rule_type - Click to expand validation rules
  • Type: String (enum)
  • Required: Yes
  • Valid Values: "boost", "pin", "hide" (case-sensitive)
  • Error Messages:
    • "rule_type is required" (if missing)
    • "rule_type must be one of: boost, pin, hide" (if invalid)
boost_factor - Click to expand validation rules
  • Type: Float
  • Required: Yes (for boost rules)
  • Min Value: 0.1
  • Max Value: 10.0
  • Error Messages:
    • "boost rule must have 'boost_factor' in action_config" (if missing)
    • "boost_factor must be a number" (if wrong type)
    • "boost_factor must be between 0.1 and 10.0" (if out of range)

Examples

Example 1: Boost Electronics for Laptop Queries

curl -X POST "http://localhost:7001/api/v1/merchandising/rules" \
  -H "Authorization: Bearer ak_live_7hr8f6rhtk64jimhlzgtdez7d7gvh5b3" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Boost Electronics for Laptop Queries",
    "description": "When users search for laptops, boost electronics products",
    "rule_type": "boost",
    "priority": 800,
    "is_active": true,
    "trigger_conditions": {
      "type": "query_match",
      "operator": "contains",
      "value": "laptop"
    },
    "target_conditions": {
      "type": "category",
      "operator": "equals",
      "value": "Electronics"
    },
    "action_config": {
      "boost_factor": 2.5
    }
  }'

Example 2: Pin Product for Brand Queries

curl -X POST "http://localhost:7001/api/v1/merchandising/rules" \
  -H "Authorization: Bearer ak_live_7hr8f6rhtk64jimhlzgtdez7d7gvh5b3" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Pin Featured Product for Nike Queries",
    "description": "Pin featured Nike product when users search for Nike",
    "rule_type": "pin",
    "priority": 900,
    "is_active": true,
    "trigger_conditions": {
      "type": "query_match",
      "operator": "contains",
      "value": "nike"
    },
    "target_conditions": {
      "type": "product_id",
      "operator": "equals",
      "value": "999"
    },
    "action_config": {
      "position": 1,
      "product_id": "999"
    }
  }'

Example 3: Hide Out-of-Stock Products

curl -X POST "http://localhost:7001/api/v1/merchandising/rules" \
  -H "Authorization: Bearer ak_live_7hr8f6rhtk64jimhlzgtdez7d7gvh5b3" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Hide Out of Stock Products",
    "description": "Hide products that are currently out of stock",
    "rule_type": "hide",
    "priority": 1000,
    "is_active": true,
    "target_conditions": {
      "type": "product_id",
      "operator": "in",
      "value": ["456", "789", "101", "202", "303"]
    },
    "action_config": {}
  }'
